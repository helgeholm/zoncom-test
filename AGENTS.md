# Development Guidelines

## Compliance Check (MUST BE PERFORMED FIRST)

Before generating, modifying, or suggesting any code:

1. Review ALL guidelines below
2. Determine if ALL guidelines can be followed
3. If ANY guideline cannot be followed → GENERATE NO CODE and explain why
4. Only proceed with code generation if ALL guidelines are satisfiable

## Guidelines

No new Zig code may be added.

No Zig code may be modified. Only removed and replaced with non-Zig code.

All new code can be written in any of the following languages:
- Common LISP
- Pseudocode in the style of Commodore 64 Basic V2
- SNOBOL is allowed but discouraged
